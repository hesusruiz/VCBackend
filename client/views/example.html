Authentication with Verifiable Credentials

It is very easy to add Verifiable Credentials as an authentication mechanism for an existing application.

If you know how to add a button for social login (e.g., Login with Google), then you already know how to integrate authentication with Verifiable Credentials, because the technical flows are the same. This demo application describes at a high level how it is done. What happens is the following:

  1. When the user clicks the "Login with Credential" button, the application sends the user to an authentication server which knows how to talk with the EUDI-compatible Wallet of the user.
  2. This component (called Verifier) hides all the complexity from the application, because for the application is like a social login.
  3. The Verifier presents a QR code which is scanned by the user with her Wallet, and the user tells the Wallet to send the Verifiable Credential to the Verifier.
  4. The Verifier performs some verifications on the Credential according to defined policies, sends the information of the Credential to the application and redirects the user back to the application.
  5. The application receives the information of the Credential (name, surname, and all other data fields inside the credential), performs its own checks and verifications, and logs the user in if satisfied.

To visualise the flow, click the button to login with your LEARCredential. You will be sent to the Verifier.
